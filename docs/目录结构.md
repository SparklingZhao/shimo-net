## 动态加载机制说明

### 页面路由（`home-content-loader.js`）

`welcome.html` 通过 URL query 参数 `?page=xxx` 路由到对应子页面。目标路由见上表「页面路由与源文件」；实施新目录后需在 `VIEW_CONFIG` 中增加 `suite-business`、`suite-enterprise`、`sdk-integration`，并视情况保留旧 key 做兼容或重定向。

### 区块嵌入（`data-merge-source`）

产品页和首页通过 `data-merge-source` 属性按需拉取子页面片段，统一使用**绝对路径**（如 `/pages/features/...`、`/pages/suite/...`、`/pages/sdk/...`）以兼容直接访问与 Shell 内嵌两种场景。

## 生产构建

推荐执行：

```bash
npm run build
```

等价命令：

```bash
node build.js
```

如需部署到子路径，例如 `/shimo/`：

```bash
npm run build:shimo
```

构建结果（目标，实施新命名后）：

- 复制 `css/`、`js/`、`assets/`、`docs/`、`pages/`、`tools/` 到 `dist/`
- 复制根目录兼容页面 `home.html` 到 `dist/`
- 将 `pages/` 与 `tools/` 内所有 HTML 的 `data-merge-source` 内容提前内联
- 按 `--base` 参数重写 `dist/` 内 HTML / JS 中的根路径资源引用
- 生成根目录静态页面示例：
  - `dist/welcome.html`
  - `dist/index.html`
  - `dist/suite-business.html`（来自 `pages/suite/business.html`）
  - `dist/suite-enterprise.html`（来自 `pages/suite/enterprise.html`）
  - `dist/sdk-integration.html`（来自 `pages/sdk/integration.html`）
  - `dist/collaboration.html`
  - `dist/knowledge.html`
  - `dist/about.html`
  - `dist/privacy.html`
  - `dist/service.html`
  - `dist/behavior.html`

实施时需在 `build.js` 的 `ROUTES` 中增加上述新页并视情况保留或删除旧 product 路由。

### 本地预览

先构建，再预览：

```bash
npm run build
npm run preview
```

子路径预览：

```bash
npm run build:shimo
npm run preview:shimo
```

## 迁移清单（实施新目录与命名时）

1. **新建目录与文件**：`pages/suite/business.html`、`pages/suite/enterprise.html`、`pages/sdk/integration.html`（可从现有 `office-suite-saas`、`office-suite-private`、`weboffice-sdk` 复制后改内容与引用）。
2. **home-content-loader.js**：在 `VIEW_CONFIG` 中增加 `suite-business`、`suite-enterprise`、`sdk-integration` 指向新路径；导航链接改为新 `?page=`。
3. **build.js**：在 `ROUTES` 中增加三条新路由，output 为 `suite-business.html`、`suite-enterprise.html`、`sdk-integration.html`；旧 product 路由可保留做重定向或删除。
4. **协作 / 知识 Tab**：在 `collaboration.html`、`knowledge.html` 及导航中，将 `data-tab` 与链接 `?tab=` 改为 document、writer、sheet、slide、table、form 与 portal、space、knowledge-base；同步修改 `doc-tabs.js`、`knowledge-tabs.js` 的默认值与兼容逻辑（若有）。
5. **企业管理**（可选）：若采用 `management-panel` 路由，将 `enterprise.html` 重命名为 `management-panel.html` 并更新路由与导航。
